# Илмий Кенгаш — Бот для тайного голосования

Telegram-бот с Mini Web App для проведения тайного голосования на заседаниях учёного совета.

## Структура проекта

```
voting_bot/
├── bot.py                 # Главный файл — запуск бота
├── config.py              # Настройки (токен, ID админа)
├── database.py            # Работа с JSON-хранилищем
├── handlers_admin.py      # Команды и логика админа
├── handlers_voter.py      # Команды и логика голосующих
├── report.py              # Генерация отчёта (DOCX)
├── requirements.txt       # Зависимости Python
├── Procfile               # Для деплоя на Koyeb
├── webapp/
│   └── index.html         # Mini Web App — админ-панель
└── data/                  # Папка для JSON-файлов (создаётся автоматически)
```

## Установка и запуск

### 1. Создать бота
- Написать @BotFather в Telegram → `/newbot`
- Получить токен
- Узнать свой Telegram ID: написать боту @userinfobot

### 2. Деплой на Koyeb (рекомендуется)
1. Залить проект на GitHub
2. На koyeb.com → Create Service → GitHub
3. Указать **только 2 переменные окружения**:
   - `BOT_TOKEN` = токен от BotFather
   - `ADMIN_ID` = ваш Telegram ID
4. Всё! Бот сам:
   - определит URL на Koyeb (через `KOYEB_PUBLIC_DOMAIN`)
   - установит webhook автоматически
   - запустит веб-сервер для Mini Web App
5. После деплоя: в @BotFather → `/mybots` → Menu Button →
   указать URL: `https://ваш-домен.koyeb.app/webapp`

### 3. Локально (для тестирования)
```bash
pip install -r requirements.txt
export BOT_TOKEN="your-token"
export ADMIN_ID="your-telegram-id"
python bot.py
```
В локальном режиме бот работает через polling.
Mini Web App недоступна (нужен HTTPS), но команды бота можно тестировать.

## Как работает

### Для админа/секретаря:
1. Открывает Mini Web App (кнопка меню бота)
2. Регистрирует участников (ФИО + уникальный PIN)
3. Создаёт заседание (дата + номер протокола)
4. Внутри заседания создаёт сеансы (напр. "Диссертация Иванова")
5. В каждом сеансе добавляет вопросы для голосования
6. Отмечает присутствующих
7. Запускает голосование по сеансу
8. После завершения — скачивает отчёт

### Для участников:
1. Первый раз: `/start` → вводит PIN → привязка к профилю
2. Когда голосование активно: получает inline-кнопки "За / Против / Воздержался"
3. После голосования — подтверждение

### Анонимность:
- Бот записывает ЧТО проголосовал (за/против/воздержался) БЕЗ привязки к имени
- Отдельно записывает КТО проголосовал (для контроля кворума)
- Связать голос с человеком невозможно
